HackerDojo - Room Reservation App

Worked with the dev team at Hacker Dojo to implement a conference room reservation application that could be used by Hacker Dojo members. I was responsible to implement the front end which would communicate with the back end (developed in Phyton) to store slots reservation.

The following features were implemented:

  • Display a responsive weekly calendar (for tablets and desktops) and a daily calendar (for small devices)
  • Populate the calendar with slots reserved by HackerDojo users. Slots information is stored in "..." DB and sent to the web browser in JSON format via API developed in Phyton (Ajax call is made to get slots information).
  • Provide the ability to reserve slots up to 30 days from current date (an Ajax call is made when user clicks on a specific slot to reserve it).
  • Provide the ability to delete a slot by just clicking over the slot cell (an Ajax call is made to delete a slot when the user clicked on it).
  • Update style of slot when it is reserved or deleted.
  • Display previous and next buttons to navigate to different weeks (for weekly calendar view) or different days (for daily calendar view). User should not be able to navigate more than 30 days from today.
  • Display sign-in screen if hackerDojo user is not logged-in with his/her HackerDojo email address.



Insurance Selector Tool (Class Project)

This Insurance Selector Tool was developed for the JQuery class I took at UC Santa Cruz Extension. The project had the following requirements:

  • Create a sample insurance picker tool single page application that will let users to view different plans and select one.
  • The application will provide suitable different viewing options. It will contain the following screens:

    • Screen1: The screen will provide one of the two options, Limited and Comprehensive coverage. The user will choose one of the two options by clicking “Select” button for the appropriate section.
    • Screen2: Clicking the select button in screen 1 will display the screen 2 without reloading the page. Screen 2 will show all individual plans for the selected coverage.
    • Screen3: Screen 3 will display a form for users to enter their personal details. Users can add information for multiple persons by clicking “Add more insured” link. The form will also have standard required field check.
    • Screen4: Upon successful submission of the form, user will get a “Thank you” page (screen 4) as a confirmation. The content should include thank you text addressed to user with their first name.

    • The following non-mandatory features were also implemented:
      • Implement breadcrumps (to show users the flow of the screens).
      • If the browser “back button” is clicked, it should display the view that was displayed previously without causing a page reload.
      • Add the ability to delete insured use(s).
